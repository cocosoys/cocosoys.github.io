开发工坊笔记

发表于2026-05-05|更新于2026-05-05|工坊日志
|浏览量:

这个站点会从“默认博客”慢慢变成一个更像工坊的地方。

我希望它能同时放下几类内容：插件开发里的坑、旧版本环境的兼容经验、工具脚本的整理，以及一些偏 ACG 氛围的创作实验。它不需要像企业官网那样端着，也不需要变成满屏角色图。更理想的状态是：打开首页时能知道这是 Cocosoys 的项目空间，继续往下读时又能找到实际可用的项目和笔记。

为什么叫工坊

工坊意味着东西还在打磨。项目可以小，但要有清晰边界；文章可以是手记，但要说明思路；视觉可以有二次元气质，但不能盖过内容。

我会优先整理什么

  1. 项目说明：把 GitHub 仓库从链接列表整理成项目地图。
  2. 开发笔记：记录插件、命令、接口和构建过程里的具体问题。
  3. 创作设定：保留一点轻量的世界观和视觉气质，让站点不是纯工具箱。

这也是之后写文章的标准：少一点模板话，多一点真实上下文。

我的 GitHub 项目地图
这篇不是完整仓库索引，而是一张“我在做什么”的项目地图。比起把链接全部堆出来，我更想按用途整理：哪些是服务器工具，哪些是玩法扩展，哪些是创作实验，哪些是在旧环境里补接口。 服务器工具 AgainRunCommand：把指令之间的联动规则配置出来，适合整理服务器里的重复流程。 CheckCommandSource：查询注册指令来自哪个插件，排查命令冲突时更直观。 AutoSellForPlayer：围绕玩家捡起物品后的自动出售做流程简化。 HealthCMD：通过命令按百分比控制玩家生命值，偏管理工具。 玩法扩展 SXAttributeExtraDigRange：给 SX 属性增加 DigRange，让工具拥有范围破坏方块的能力。 DoubleMineral：一个简单的时运方块效果实验，主要面向旧版本环境。 兼容与接口 CValue：围绕 JRMCore 提供补充 API，重点是让已有生态更容易接入。 创作实验 ArcaneResearch：尝试把研究笔记、解谜和世界观组合成一种可继续扩展的内容系统。 下一步整理方向我会优先把这些项目的使用场景、版本边界和配置样例补齐。真正...
